SDK Documentation

Complete guide to integrating the SMATCH Licensing SDK into your application. Secure your software with robust license validation, machine binding, and offline support.

Secure Validation

Validate licenses online with API key authentication and rate limiting.

Offline Support

Validate licenses without internet using HMAC checksums and signatures.

Machine Binding

Restrict licenses to specific machines using MAC address binding.

Cryptographic Security

HMAC checksums and NaCl signatures ensure license integrity.

Installation #

Install the SDK using pip:

bash 
pip install smatch-licensing-api-client

Or install from source:

bash 
pip install -e ./licensing-api-client

Requirements: Python 3.8+ and the requests library. The pynacl library is required for signature verification.

Quick Start #

Get started with just a few lines of code:

python 
from smatch_licensing_api_client import LicenseClient

# Initialize the client
client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key_here"
)

# Validate a license
is_valid = client.validate("XXXXX_XXXXX_XXXXX_XXXXX_XXXXX")
if is_valid:
    print("License is valid!")
else:
    print("License is invalid!")

API Key Authentication #

All requests to the licensing API require an API key. You can create API keys from the dashboard.

Security: Never expose your API secret key in client-side code. Only the public API key should be used in your applications.

Client Configuration

python 
from smatch_licensing_api_client import LicenseClient

# Basic configuration
client = LicenseClient(
    server_url="http://localhost:8000",  # Your licensing server URL
    api_key="sk_your_api_key",           # Public API key
    secret_key="your_secret_key",          # Secret key for offline HMAC
    public_key="your_public_key"          # Public key for signature verification
)
Parameter Type Required Description
server_url string Yes Base URL of your licensing server
api_key string Yes Public API key from dashboard
secret_key string No Secret key for offline HMAC validation
public_key string No Public key for cryptographic signatures

License Validation #

Validate licenses online against the licensing server. This is the most secure method as it checks the server's database directly.

Your App Calls validate()
→
SDK Sends API request
→
Server Checks database
→
Response Valid/Invalid

Online Validation

python 
from smatch_licensing_api_client import LicenseClient

client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key"
)

# Validate a license key
result = client.validate("XXXXX_XXXXX_XXXXX_XXXXX_XXXXX")

if result.is_valid:
    print(f"License is valid!")
    print(f"Expires: {result.expires_at}")
    print(f"State: {result.state}")
else:
    print(f"License invalid: {result.reason}")

The validate() method returns a LicenseResponse object with is_valid, reason, state, and expires_at properties.

Validation Response

Property Type Description
is_valid bool Whether the license is valid
reason string Reason if invalid (e.g., "expired", "revoked")
state string License state: "active", "inactive", "suspended", "revoked"
expires_at datetime License expiration date

License Activation #

Activate a license to bind it to a machine. This is required before the license can be used.

When a license is activated, it's automatically bound to the current machine's MAC address. The max machines setting controls how many machines can bind to the license.

python 
from smatch_licensing_api_client import LicenseClient

client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key"
)

# Activate a license
# This binds the license to the current machine
success = client.activate("XXXXX_XXXXX_XXXXX_XXXXX_XXXXX")

if success:
    print("License activated successfully!")
else:
    print("Activation failed")

Activation Flow

1

Your App Calls activate()

SDK sends the license key with the machine's MAC address

2

Server Validates

Checks if license exists, is not revoked, and has available machine slots

3

Machine Binding

MAC address is added to the license's allowed machines

4

Activation Complete

License state changes to "active"

Offline Licensing #

Validate licenses without an internet connection. This is essential for desktop applications that need to work offline.

Security Note: Offline validation relies on cryptographic checksums and signatures. Ensure your secret key remains secure.

Saving License for Offline Use

python 
from smatch_licensing_api_client import LicenseClient

client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key",
    secret_key="your_secret_key",  # Required for offline HMAC
    public_key="your_public_key"    # Required for signature verification
)

# First, validate online
if client.validate("XXXXX_XXXXX_XXXXX_XXXXX_XXXXX"):
    # Save license to file for offline use
    client.save_license("XXXXX_XXXXX_XXXXX_XXXXX_XXXXX", "license.dat")
    print("License saved for offline use!")

Verifying Offline License

python 
from smatch_licensing_api_client import LicenseClient

client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key",
    secret_key="your_secret_key",
    public_key="your_public_key"
)

# Verify license file (offline)
# This checks HMAC checksum and NaCl signature
is_valid = client.verify_offline("license.dat", check_signature=True)

if is_valid:
    print("Offline license is valid!")
else:
    print("License verification failed - may be tampered or expired")

Offline Validation Process

Load File Read license.dat
→
HMAC Check Verify checksum
→
Signature Verify NaCl sig
→
Expiry Check expiration

Machine Binding #

Restrict licenses to specific machines using MAC address binding. This prevents license sharing.

How It Works

  1. Each machine has a unique MAC address
  2. When activating, the MAC is bound to the license
  3. License validates by checking the machine's MAC against allowed list
  4. Max machines setting controls how many devices can use the license

Note: You can set max machines when creating a license or update it later from the dashboard. Use -1 for unlimited machines.

Custom MAC Address

python 
from smatch_licensing_api_client import LicenseClient, get_mac_address

client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key"
)

# Auto-detect MAC address
mac = get_mac_address()
print(f"Machine MAC: {mac}")

# Or specify a custom MAC
client.activate("XXXXX_XXXXX_XXXXX_XXXXX_XXXXX", mac_address="AA:BB:CC:DD:EE:FF")

HMAC Checksum #

HMAC (Hash-based Message Authentication Code) ensures license data integrity. The secret key is used to generate a checksum that can't be forged without the key.

python 
# HMAC is automatically computed when saving/verifying offline licenses
# The license file contains:
# {
#     "license_key": "XXXXX_XXXXX_XXXXX_XXXXX_XXXXX",
#     "expiry_date": "2026-04-03T22:46:01",
#     "checksum": "a1b2c3d4...",  # HMAC-SHA256 of the data
# }

# To verify manually:
import hmac
import hashlib

def verify_checksum(data, checksum, secret_key):
    expected = hmac.new(
        secret_key.encode(),
        data.encode(),
        hashlib.sha256
    ).hexdigest()
    return hmac.compare_digest(expected, checksum)

Warning: Anyone with the secret key can generate valid checksums. Keep your secret key secure and never distribute it in client-side code for production.

NaCl Signatures #

For enhanced security, use Ed25519 digital signatures from the NaCl library. This provides asymmetric cryptography where only the server can sign licenses.

python 
from smatch_licensing_api_client import LicenseClient

client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key",
    public_key="53b12a371b91d92c80d4afb8c12bbc3af00c1b426c4f0f1a78f1eac2239bfd9c"
)

# When verifying offline, enable signature check
is_valid = client.verify_offline(
    "license.dat",
    check_signature=True  # Requires public_key to be set
)

if is_valid:
    print("License signature verified!")

Recommendation: Always enable signature verification in production. The public key can be safely distributed in your application.

Error Handling #

The SDK provides specific exception classes for different error scenarios:

python 
from smatch_licensing_api_client import LicenseClient
from smatch_licensing_api_client.errors import (
    LicenseError,
    LicenseInvalidError,
    LicenseExpiredError,
    LicenseRevokedError,
    LicenseNotFoundError,
    NetworkError,
    AuthenticationError
)

client = LicenseClient(
    server_url="http://localhost:8000",
    api_key="sk_your_api_key"
)

try:
    result = client.validate("XXXXX_XXXXX_XXXXX_XXXXX_XXXXX")
except LicenseNotFoundError:
    print("License not found")
except LicenseExpiredError:
    print("License has expired")
except LicenseRevokedError:
    print("License has been revoked")
except NetworkError:
    print("Network error - check connection")
except AuthenticationError:
    print("Invalid API key")
except LicenseError as e:
    print(f"License error: {e}")
Exception Description
LicenseNotFoundError License key doesn't exist
LicenseInvalidError License is invalid for other reasons
LicenseExpiredError License has expired
LicenseRevokedError License was revoked
LicenseTamperedError Offline license data was tampered
NetworkError Network connection failed
AuthenticationError Invalid API key
